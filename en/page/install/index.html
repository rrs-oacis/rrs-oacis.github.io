(1) Installing RRS-OACIS

Using Docker tool It is easy to introduce the virtual environment where RRS-OACIS is installed.It can be introduced not only to Linux but also Windows, macOS The outline of the procedure is as follows

The OS assumes Ubuntu

Required environment

  • Docker (checked 17.09.0-ce)
  • OS where Docker runs
  • Internet connection

Install Docker (Skip the already installed one)

Install Docker on Ubuntu, please follow the link below to install it

Install Docker on Ubuntu

* Window is From here , Mac users here Please install from

Installing RRS-OACIS

You can build an environment where RRS-OACIS was introduced by executing the following script.

※ It may take several minutes to build

Please execute on PC with Docker installed

sudo docker run --name rrsoacis -p 3080:3080 -p 3000:3000 -p 49138:49138/udp -dt rrsoacis/rrsoacis

Installation is completed if displayed as below

Digest: sha256:(※UID)
Status: Downloaded newer image for rrsoacis/rrsoacis:latest

in browser access

 http://localhost:3080/

and see if it appears as below.

(The following image can be magnified)

Tips
There is no restriction on the OS to install RRS-OACIS, you can run any OS if Docker is running.

(2) Set up the cluster

The OS assumes Ubuntu (I confirmed that it can be executed by the same operation on macOS.)

Required environment

  • Ubuntu 16.04.3 LTS
  • Oracle JDK (checked 1.8.0_40)
  • OpenSSH
  • Ubuntu's local account with password set

Go to setting screen

in the browser

    http://localhost:3080/settings-clusters

accessed

Set up Cluster

Step 1

Enter the user name and IP address of the PC to be clustered, format is" user name @ IP address".

Step 2

Sets the save format of the simulation log file Select the format supported by the cluster OS, gzip, 7 zip and zip are available There

* Ubuntu is the default gzip format and there is no problem.

Step 3

Enter the cluster password

* Password for Ubuntu's account, assuming the same password for all four.

Step 4

Add by pressing the Add button.

Step 5

Setting is completed when displayed as above.

Tips
It is possible to build with only 1 to 3 clusters, there is no need to prepare 4 clusters.

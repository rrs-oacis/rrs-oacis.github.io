Explain how to upload the original Agent to RRS-OACIS

Create

For RRS-OACIS, you can use the agent that used RoboCupRescue Simulation Agent Development Framework (RCRS-ADF) as a prerequisite. Refer to the tutorial on RCRS-ADF and create Agent.

Prepared

Once you have created the Agent, prepare an Agent Zip file with the following configuration

※ Zip file name is optional

  • src
    • rescue code...
  • config
    • module.cfg
  • data
    • develop.json
  • library
    • rescue
      • adf
        • adf-core.jar
      • rescuecore
        • clear.jar
        • collapse.jar
        • gis2.jar
        • handy.jar
        • human.jar
        • ignition.jar
        • kernel.jar
        • maps.jar
        • misc.jar
        • rescuecore2.jar
        • resq-fire.jar
        • sample.jar
        • standard.jar
        • traffic3.jar
    • util
      • commons-math3-3.4.1.jar
      • guava-20.0.jar
      • jackson-annotations-2.8.5.jar
      • jackson-core-2.8.5.jar
      • jackson-databind-2.8.5.jar
      • jackson-dataformat-msgpack-0.8.11.jar
      • jsr305-3.0.1.jar
      • msgpack-core-0.8.11.jar
  • compile.sh
  • precompute.sh
  • start.sh

Upload

Upload Agent data used for simulation to RRS-OACIS.

Move to operation screen

Click "Agents" from the sidebar.

(the following images can be magnified)

Or, to the browser,

http://localhost:3080/agents

to access

Operation

Step 1

Select the Agent to upload

Step 2

Enter name of Agent

You can use alphanumeric characters and underscores in the name

Step 3

Press the button to upload Agent data to RRS-OACIS.

Step 4

Upload completed when displayed as above.

Add Cluster

Video

Explain how to increase the number of clusters used in simulation.

The OS assumes Ubuntu (I confirmed that it can be executed by the same operation on macOS.)

Required environment

  • Ubuntu 16.04.3 LTS
  • Oracle JDK (checked 1.8.0_40)
  • OpenSSH
  • Ubuntu's local account with password set

Go to setting screen

in the browser

    http://localhost:3080/settings-clusters

Access

Setting up Cluster

Step 1

Enter the user name and IP address of the PC you want to cluster, in the format "user name @ IP address".

Step 2

Sets the saving format of the simulation log file Select the format supported by the cluster OS

* Ubuntu is the default gzip format and there is no problem.

Step 3

Enter the cluster password

※ Password of Ubuntu account

Step 4

Add by pressing the Add button.

Step 5

Setting is completed when displayed as above.

Tips
It is also possible to build one cluster with only 1 to 3, there is no need to prepare 4 sets.

オリジナルのAgentをアップロードする

RRS-OACISにオリジナルのAgentのアップロード方法を説明します.

作成

RRS-OACISではRoboCupRescue Simulation Agent Development Framework (RCRS-ADF)を前提に使用したAgentを使うことができます． RCRS-ADFのチュートリアルを参考にAgentの作成をしてください.

用意

Agentの作成をしたら下記のような構成のAgentのZipファイルを用意します.

※Zipのファイル名は任意です.

  • src
    • rescue code...
  • config
    • module.cfg
  • data
    • develop.json
  • library
    • rescue
      • adf
        • adf-core.jar
      • rescuecore
        • clear.jar
        • collapse.jar
        • gis2.jar
        • handy.jar
        • human.jar
        • ignition.jar
        • kernel.jar
        • maps.jar
        • misc.jar
        • rescuecore2.jar
        • resq-fire.jar
        • sample.jar
        • standard.jar
        • traffic3.jar
    • util
      • commons-math3-3.4.1.jar
      • guava-20.0.jar
      • jackson-annotations-2.8.5.jar
      • jackson-core-2.8.5.jar
      • jackson-databind-2.8.5.jar
      • jackson-dataformat-msgpack-0.8.11.jar
      • jsr305-3.0.1.jar
      • msgpack-core-0.8.11.jar
  • compile.sh
  • precompute.sh
  • start.sh

アップロード

シミュレーションで使用するAgentデータをRRS-OACISにアップロードします．

操作画面に移動

サイドバーから「Agents」をクリックします.

（以下の画像は拡大出来ます）

または，ブラウザに 

http://localhost:3080/agents

と入力してアクセスします.

操作

ステップ1

アップロードするAgentを選択します.

ステップ2

Agentの名前を入力します.

※名前には英数字とアンダースコアが使用できます

ステップ3

AgentデータをRRS-OACISにアップロードするためにボタンを押します.

ステップ4

上のように表示されたらアップロード完了です.

または，Agentの画面からLinkで移動できます.

ステップ1

ステップ2

Agentのデータがダウンロードできます.

クラスターを追加

シミュレーションで使用するクラスターを増やす方法を説明します．

OSはUbuntuを想定して説明します．(macOSでも同様の操作で実行できることを確認しました．)

必須環境

  • Ubuntu 16.04.3 LTS
  • Oracle JDK (checked 1.8.0_40)
  • OpenSSH
  • パスワードの設定されたUbuntuのローカルアカウント

設定画面に移動

ブラウザで

    http://localhost:3080/settings-clusters

にアクセスします.

クラスターの設定

ステップ1

クラスターにするPCのユーザー名とIPアドレスを入力します．形式は「ユーザー名@IPアドレス」になります．

ステップ2

シミュレーションのログファイルの保存形式を設定します．クラスターのOSが対応している形式を選択します．

※Ubuntuの場合はデフォルトのgzip形式で問題ありません．

ステップ3

クラスターのパスワードを入力します．

※Ubuntuのアカウントのパスワードです．

ステップ4

Addボタンを押して追加します．

ステップ5

上のように表示されたら設定完了です．

Tips
一つのクラスターを1~3台だけで構築する事も可能です．4台用意する必要はありません．
